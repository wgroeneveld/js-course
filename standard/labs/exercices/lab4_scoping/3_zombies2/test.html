Lab 3: Zombies (revisited) are not eating politely

So you've reduced an army to dust... But you forgot a few hungry zombies, and now they're eating stuff! Waargh!

A zombie will 'eat' anything which stands in their way - plants, strings, primitives, you name it.
Zombies don't like plain food, they like it with a lot of sauce! And they are not very polite eaters. Zombies can slurp, swallow whole chunks and feast upon things.

You'll need to build a zombie function which allows a zombie to eat ONE thing, with multiple sauces, on multiple "politeness" levels. This function will behave similar to a builder pattern. Instead of calling 'build()', you can use 'swallow()'.


This is a concrete example: 
var unpoliteMeal = zombie()
	.eat("pumpkin plant")
	.with("extra virgin olive oil")
	.slurping()
	.feastingUpon()
	.swallow();
// unpoliteMeal === "zombie slurps and feats upon pumpkin plant with extra virgin olive oil"

// public interface:
var edible = {
	with,
	slurping,
	swallow,
	feastingUpon
}

function zombie() {
	return {
		eat: edible
	};
}

You can mix and match function calls! like so: 

var unpoliteMeal = zombie().eat("pumpkin").slurping().with("mayo").feastingUpon().with("oil");
// unpoliteMeal === "zombie slurps and feats upon pumpkin with mayo and oil

Different things will be separated with 'and' for politeness level and sauces. Don't forget to add the "with" word while appending the sauces.
. You should NOT be able to eat two things at once! So .eat("stuff").eat will be undefined.
If you're not sure what to do, please take a look at the tests for some examples.

Another tip: remember 'reduce()'?
